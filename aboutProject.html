AndroidMUI5 · Jetchat Extensions

SMS Inbox & Voice-to-Text Playground

AndroidMUI5 extends Google's Jetchat sample with multiple SMS views and a full Voice-to-Text lab (V1–V4) built with Jetpack Compose, Material 3 and Android's SpeechRecognizer API.

What this project demonstrates

A real-world style app that shows how to combine classic chat UI, experimental SMS layouts and rich speech input experiences – all under one consistent navigation shell.

Jetpack Compose Material 3 Navigation Drawer SpeechRecognizer Kotlin

Feature timeline

Base Jetchat
Original chat sample with profiles, conversations and widget integration.
SMS Inbox V1–V5
Multiple list + detail styles to explore how conversations can be presented.
Voice to Text V1–V4
From a simple dictation pad to live streaming text and edit-able chat bubbles.

All versions are accessible from a collapsible drawer: Chats, SMS, and Voice to Text.

Voice to Text versions

V1 · Minimal Dictation

A single button to start / stop listening and one large area to collect all text. Perfect when you just want a clean & simple “notes from voice” screen.

V2 · Session Layout

Session card + transcript card, clearly separating the recording controls from the resulting text. Designed for more formal or clinical dictation scenarios.

V3 · Live + History

Shows partial results in a live area while finalized sentences accumulate below. A floating mic button makes it feel like a recorder.

V4 · Chat Bubbles

Every sentence becomes a message bubble. Edit or delete individual bubbles and copy all with one tap – ideal for drafting messages by voice.

SMS Inbox experiments

Multiple SMS Inbox versions coexist side-by-side. Each version explores different list densities, card shapes and detail views. The collapsible “SMS” group in the drawer lets you quickly switch between them for UI comparison or user testing.

For developers

The project is structured for easy extension. To add a new menu entry in the drawer, you typically:

  1. Add a label in strings.xml.
  2. Create a new DrawerDestination object.
  3. Include it in the drawer grouping (Chats / SMS / Voice to Text).
  4. Implement a new composable screen under feature/.
  5. Wire it into the when(selectedDestination) block in NavActivity.

A dedicated developer guide in the repo walks through this step by step.
