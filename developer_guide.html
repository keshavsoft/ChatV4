Developer Guide · Add New Menu in Sidebar

Visual walkthrough of DEVELOPER_GUIDE.md for AndroidMUI5 (Jetchat + SMS + Voice to Text).

Navigation Drawer · Jetpack Compose

Overview

The left sidebar (drawer) is powered by a sealed class DrawerDestination and used in three main places:

core/navigation/DrawerDestination.kt
components/JetchatDrawer.kt
NavActivity.kt

Whenever you want to add a new item in the drawer (for example “Reports” or “VoiceToTextV5”), you will follow the same pattern:

1
Add a label in strings.xml
File: app/src/main/res/values/strings.xml

Add a new string resource for the menu item text that will be shown in the drawer:

<string name="menu_reports">Reports</string>

Naming pattern: menu_* keeps all drawer labels grouped together.

2
Create a new DrawerDestination
File: core/navigation/DrawerDestination.kt

Inside the sealed class, add a new object for your destination:

data object Reports : DrawerDestination(
  "Reports",
  R.string.menu_reports
)

Then register it in the fromKey function so the app can map a key string back to this destination:

companion object {
  fun fromKey(key: String): DrawerDestination = when (key) {
    Composers.key -> Composers
    Gps.key -> Gps
    Reports.key -> Reports // new
    else -> Composers
  }
}
3
Place it into a drawer group
File: components/JetchatDrawer.kt

The drawer is divided into groups like Chats, SMS and Voice to Text. To show your new menu item, add it to the matching list.

// Chats section
DrawerItemHeader("Chats")

val mainChatItems = listOf(
  DrawerDestination.Composers,
  DrawerDestination.TestByKeshav,
  DrawerDestination.Droidcon,
  DrawerDestination.Gps,
  DrawerDestination.Reports // <-- new
)

If it belongs under SMS or Voice to Text, add it there instead of mainChatItems.

4
Create the Composable screen
File: feature/reports/ReportsScreen.kt (example)

Every menu destination simply renders a composable screen. The first version can be very small, you can enhance it later.

@Composable
fun ReportsScreen(onBack: () -> Unit = {}) {
  Scaffold(
    topBar = {
      CenterAlignedTopAppBar(
        title = { Text("Reports") }
      )
    }
  ) { innerPadding ->
    Box(Modifier.padding(innerPadding)) {
      Text("Reports screen coming soon…")
    }
  }
}
5
Wire it in NavActivity
File: NavActivity.kt

NavActivity decides which screen to show based on the current selectedDestination. Add your new destination to this when block.

when (selectedDestination) {
  DrawerDestination.Sms -> { /* SMS screens */ }
  DrawerDestination.VoiceToTextV4 -> { /* V4 screen */ }
  DrawerDestination.Reports -> {
    ReportsScreen(onBack = {
      selectedDestination = DrawerDestination.Composers
    })
  }
  else -> { AndroidViewBinding(ContentMainBinding::inflate) }
}

Don’t forget the import at the top: import com.example.compose.jetchat.feature.reports.ReportsScreen

Quick reference

Files you will touch
  • res/values/strings.xml
  • core/navigation/DrawerDestination.kt
  • components/JetchatDrawer.kt
  • feature/<yourFeature>/YourScreen.kt
  • NavActivity.kt
Typical pattern
  1. Define label in XML
  2. Define DrawerDestination object
  3. Add to drawer list
  4. Create composable screen
  5. Add a when branch in NavActivity

Checklist before commit

New <string> added with a unique name.
DrawerDestination object created and registered in fromKey().
Destination included in the right drawer group list.
Composable screen file compiles without errors.
NavActivity shows your screen in the when block.

Example: adding VoiceToTextV5

To add another VoiceToText version, you repeat the same flow:

  1. Add menu_voice_to_text_v5 in strings.xml.
  2. Add data object VoiceToTextV5 to DrawerDestination.
  3. Include VoiceToTextV5 in the Voice group list in JetchatDrawerContent.
  4. Create VoiceToTextScreenV5.kt in feature/voicetotext.
  5. Add a branch in NavActivity that calls VoiceToTextScreenV5.

This keeps all Voice versions consistent, just like V1–V4 already implemented in the project.
